# Готовые промпты для ИИ-агента

Копируй и подставляй значения в `{...}`. Общие правила работы агент берёт из `CLAUDE.md` автоматически.

## 1. Kickoff (первая сессия)

```
Прочитай CLAUDE.md, docs/00-vision.md, docs/01-architecture.md, docs/02-stack.md,
docs/03-repo-structure.md, docs/04-conventions.md и docs/plan/ROADMAP.md.

Начинаем фазу 0. Выполни задачи 0.1 и 0.2:
- скаффолдинг монорепозитория строго по docs/03-repo-structure.md;
- compose.dev.yaml с инфраструктурой; .env.example; health-эндпоинты.

Сначала покажи короткий план (структура, ключевые конфиги), затем реализуй.
В конце: pnpm typecheck && pnpm lint зелёные, инструкция запуска в README апдейтом,
обнови docs/plan/ROADMAP.md (чекбоксы) и docs/plan/STATUS.md (журнал сессии).
```

## 2. Рабочая сессия (шаблон)

```
Прочитай docs/plan/STATUS.md и docs/plan/ROADMAP.md — мы в фазе {N}.
Прочитай спеку: docs/{путь к спеке модуля}.

Выполни задачи {N.x}–{N.y}.
Требования:
- строго по спеке; неясности решай минимально и фиксируй в STATUS.md → «Принятые решения»;
- Definition of Done из CLAUDE.md §4 для каждой задачи;
- UI сверь с чеклистом docs/06-design-system.md §8.

Сначала план (файлы/миграции/эндпоинты/компоненты), потом код.
В конце — отметь чекбоксы в ROADMAP.md и запись в STATUS.md.
```

## 3. Продолжение прерванной сессии

```
Прочитай docs/plan/STATUS.md (последняя запись журнала) и выполни git status/log за сегодня.
Определи, на чём остановились, кратко доложи и продолжи незавершённую задачу {N.x}.
Ничего не переписывай с нуля без необходимости.
```

## 4. Код-ревью фазы (перед приёмкой)

```
Проведи ревью изменений фазы {N} (git diff {tag/commit}..HEAD) по критериям:
1. Соответствие спеке docs/{спека} — перечисли расхождения.
2. Безопасность: проверки прав на каждом эндпоинте, валидация, docs/09-security.md §1–3.
3. Конвенции docs/04: формат ошибок, пагинация, i18n, миграции.
4. UI: чеклист docs/06 §8 по каждому новому экрану.
5. Тесты: покрыты ли критерии приёмки спеки.
Выдай список: [критично / желательно / мелочь] с файлами и строками. Ничего не чини без команды.
```

## 5. Багфикс

```
Баг: {описание. что делал / что ожидал / что получил. скриншот/лог если есть}.
Найди причину (прочитай связанный код и спеку модуля), объясни в 2–4 предложениях,
предложи фикс, после подтверждения — реализуй + регрессионный тест.
```

## 6. Приёмка фазы (прогон критериев)

```
Мы завершаем фазу {N}. Открой раздел «Критерии приёмки» в docs/{спека модуля}
и docs/plan/ROADMAP.md → «Приёмка» фазы.
Пройди по каждому пункту: покажи, чем он подтверждается (тест/скрин/команда).
Пункты без подтверждения — доделай или занеси в STATUS.md «Известные проблемы» с [P1].
```

## Советы по работе с агентом

- **Одна сессия — 1–3 задачи чеклиста**, не больше: качество падает на длинных сессиях.
- Просить **план до кода** — дёшево отсеивает неверное понимание.
- Коммитить после каждой принятой задачи; фазу завершать git-тегом `phase-{N}`.
- Раз в фазу запускать промпт №4 (ревью) в **свежей сессии** — свежий контекст находит больше.
- Если агент «поплыл» (переписывает лишнее, спорит со спекой) — прервать, начать новую сессию с промпта №3.
- Все «хотелки» вне спек — сначала записать в спеку (или STATUS «Решения»), потом просить реализацию.
