# Модуль 13. Чат

Внутренний мессенджер: каналы, личные и групповые сообщения, упоминания, файлы, поиск. Замена WhatsApp/Telegram в служебной переписке.

## 1. Права
`chat.use` (все) · `chat.channels.create` · `chat.admin` (модерация всего, экспорт). Роли в канале: owner / admin / member (member_role в chat_members).

## 2. Типы бесед
- **Публичный канал** — виден в каталоге, вступление свободное.
- **Закрытый канал** — по приглашению; невидим в каталоге.
- **Канал подразделения** — создаётся автоматически для каждого org_unit (члены = сотрудники, синхронизация при кадровых изменениях).
- **Канал ЧС** — создаётся из карточки ЧС кнопкой (имя `чс-{номер}`, связь entity_links, участники — по матрице оповещения + вручную).
- **ЛС (DM)** и **групповые ЛС** (до 20 человек, без имени — список участников).

## 3. Модель данных (`app`)
**chat_channels**: kind `public|private|org|incident|dm|group`, name null, topic null, org_unit_id null, created_by, is_archived, last_message_at. **chat_members**: channel_id, user_id, member_role, joined_at, last_read_message_id, notify_level `all|mentions|mute`, is_pinned (личная закладка). **chat_messages** (партиции по месяцам): channel_id, author_id null (system), kind `text|file|system|call`, body (TipTap JSON, ≤ 4000 симв.), reply_to_id null, file_ids uuid[], created_at, edited_at, deleted_at (заглушка «удалено»), FTS по извлечённому тексту. **chat_reactions**: message_id, user_id, emoji (уникальность тройки). **chat_pins**: channel_id, message_id, pinned_by.

## 4. Функциональность
- Отправка: Enter (Shift+Enter — перенос), markdown-lite через RichTextEditor (жирный/курсив/код/списки/ссылки), **упоминания** `@имя` (поиск по людям канала) и `@channel` (только admin канала), вставка скриншота из буфера, drag-n-drop файлов (через files-модуль, space=system).
- Реплаи (цитата-ссылка), реакции (палитра из 20 эмодзи + недавние), редактирование 24 ч (метка «изменено»), удаление своих (админ — любых), закреплённые сообщения (панель).
- Непрочитанные: разделитель «Новые», бейджи в сайдбаре канала и глобальном сайдбаре, счётчик упоминаний отдельно (красный). `last_read` обновляется по видимости последнего сообщения.
- Typing-индикатор (WS, throttle 3с), presence (онлайн/away 10 мин/офлайн — Redis TTL), статус в аватарах.
- Поиск: по текущему каналу и глобально (FTS), фильтры от:@user, в:#канал, период; результат — прыжок к сообщению с контекстом.
- Системные сообщения: вступления, звонки (`kind=call` — карточка звонка с длительностью и участниками), закрепы.
- Ссылки на сущности платформы (`/app/incidents/…`) разворачиваются в EntityLink-карточку.

## 5. Realtime-протокол (Socket.IO)
Комнаты `channel:{id}`; вступление при открытии канала (сервер проверяет членство). События: `chat.message.created|updated|deleted`, `chat.reaction.updated`, `chat.typing`, `chat.read` (для галочек «прочитано» в DM), `chat.channel.updated`, `presence.changed`. Мутации — REST; optimistic UI: сообщение рендерится сразу со статусом «отправляется», реконсиляция по id.
Догрузка истории — cursor pagination вверх (50), якорь-скролл. При reconnect — refetch последних 50 + непрочитанных счётчиков.

## 6. Уведомления
Упоминание или DM при офлайне/не-в-канале → in-app notify мгновенно; email — если офлайн > 5 мин (digest-склейка). notify_level канала уважается. Клик по уведомлению — прыжок к сообщению.

## 7. UI (`/app/chat`)
Три колонки: (1) список бесед — поиск, секции Закреплённые/Каналы/Личные, бейджи, кнопка «+» (новый канал/ЛС, каталог публичных); (2) лента — шапка канала (имя, тема, участники-аватары, кнопки: звонок 📞, видео 📹, инфо), виртуализированный список сообщений (группировка по автору и дню), композер снизу; (3) панель инфо (по кнопке): описание, участники (+управление по роли), закреплённые, файлы канала, связанная ЧС. Мобильный: колонки становятся стеком с навигацией назад.

## 8. API (основное)
```
GET/POST /chat/channels  GET /chat/channels/catalog  PATCH /chat/channels/:id
POST /chat/channels/:id/members  DELETE .../members/:userId  POST /chat/dm {userIds}
GET /chat/channels/:id/messages?cursor  POST /chat/channels/:id/messages
PATCH/DELETE /chat/messages/:id  PUT /chat/messages/:id/reactions {emoji}
POST /chat/channels/:id/read {messageId}  GET/POST /chat/channels/:id/pins
GET /chat/search?q&channel&from&period
```

## 9. Аудит/ретеншн
Аудит: создание каналов, изменение состава, экспорт. Сообщения хранятся бессрочно (гос-архив); экспорт канала в HTML/CSV — только `chat.admin` с аудитом. Партиции по месяцам создаёт retention-job.

## 10. Критерии приёмки
- Доставка сообщения между двумя клиентами < 300 мс локальной сети; optimistic UI без дублей при плохой сети (тест reconnect).
- Лента 10k сообщений скроллится плавно (виртуализация), прыжок к найденному сообщению работает.
- Упоминание офлайн-пользователя приводит к email (maildev-тест).
- Права: не-член закрытого канала не читает сообщения ни по REST, ни по WS (e2e).
- Каналы подразделений синхронизируются при переводе сотрудника (тест).

## 11. V2+
Треды, статусы-эмодзи, пересылка, отложенная отправка, голосовые сообщения, шифрование E2E (не требуется в закрытом контуре).
