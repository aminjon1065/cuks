# Модуль 10. ГИС и аналитика (основной)

Назначение: единый источник правды о ЧС и рисках на территории — реестр ЧС, карты, слои, дашборды, отчёты. Работает и в вебе, и для профессионалов в QGIS/ArcGIS.

## 1. Роли и права
`gis.view` (все сотрудники) · `incidents.create` (дежурные) · `incidents.manage` · `gis.layers.edit` · `gis.layers.manage` · `gis.import`/`gis.export` · `gis.pg.access` · `analytics.view` · `analytics.build`. Скоуп: районные видят свой район+область (конфиг per-role), центральный аппарат — всё.

## 2. Классификатор видов ЧС (справочник-дерево, сиды)

- **Природные**: геофизические (землетрясение); геологические (оползень, сель, лавина, обвал/осыпь, просадка грунта); гидрологические (паводок, наводнение, подтопление, прорыв ледникового озера); метеорологические (ливень, град, ураганный ветер, снегопад, мороз, жара, засуха, туман); природные пожары.
- **Техногенные**: транспортные аварии (ДТП с тяжкими, авиа, ж/д), пожары/взрывы в зданиях, аварии энергосистем, аварии ЖКХ (вода/тепло/газ), аварии с химически опасными веществами, обрушения зданий/сооружений, гидродинамические аварии (прорыв дамб).
- **Биолого-социальные**: эпидемии, эпизоотии, эпифитотии, массовые отравления.

Коды стабильные (`nat.geo.landslide`), дерево 2–3 уровня, редактируется админом.

**Уровни ЧС**: объектовый / местный / региональный / республиканский / трансграничный (шкала sev-1..5, см. design-system).
**Статусы ЧС**: `reported` (донесение) → `active` (в работе) → `localized` (локализована) → `eliminated` (ликвидирована) → `closed` (закрыта, данные подтверждены). Откат разрешён с правом `incidents.manage`, с причиной.

## 3. Модель данных (схема `gis`, частично `app`)

**gis.admin_units** — адм.-терр. деление: level `region|district|jamoat`, parent_id, code, name_ru/tg, geom (MultiPolygon), population int null. Импорт из OSM/официальных шейпов скриптом `seed-geo.sh`.

**app.incidents**: number (генерируется `ЧС-{YYYY}-{seq}`), type_code (справочник), severity 1–5, status, occurred_at, reported_at, region_id/district_id/jamoat_id (авто-определение по геометрии), geom (Point или Polygon, 4326), address_text, description, source `phone|report_doc|monitoring|other`, casualties: dead int, injured int, evacuated int, affected int; damage_est numeric null (сомони), damage_note, org_unit_id (ответственное), created_by, closed_at, closed_by. FTS по number+description+address.

**app.incident_reports** (донесения/сводки по ЧС во времени): incident_id, reported_at, text, casualties-поля (снимок на момент), author_id. Карточка ЧС показывает хронологию донесений; поля карточки = последнее подтверждённое.

**app.incident_resources** (задействованные силы и средства): incident_id, kind `personnel|vehicle|equipment|aviation`, name, qty, org_text, period.

**gis.facilities** (объекты инфраструктуры): kind (справочник: школа, больница, убежище/ПВР, мост, дамба, склад, пожарное депо…), name, geom (Point), attrs jsonb (ёмкость ПВР и т.п.), org_unit_id null, is_active.

**gis.risk_zones**: hazard_code (вид ЧС), name, level 1–5, geom (MultiPolygon), attrs jsonb, source, updated_by.

**gis.layers** (реестр слоёв): slug uq, title, kind `system|imported|drawn`, geometry_type, table_name (для imported — физическая таблица `gis.l_<slug>`), style jsonb (веб-стиль), is_published_wms bool, min/max zoom, description, org_unit_id null, acl через resource_acl. Системные слои: admin_units, incidents, facilities, risk_zones.

**gis.layer_features** — для kind=drawn (нарисованные в вебе объекты): layer_id, geom, props jsonb.

**app.gis_imports**: file_id, layer_id null, status `pending|processing|done|failed`, log text, options jsonb.

## 4. Карта-эксплорер (`/app/map`) — центральный экран

- Полноэкранная MapLibre-карта, подложки: «Схема» (PMTiles Protomaps, ru-лейблы), «Тёмная», опционально «Спутник» (если админ настроит внешний XYZ). Переключатель подложек.
- **Панель слоёв** (слева, сворачиваемая): дерево групп (Оперативная обстановка / Инфраструктура / Риски / Мои слои), чекбоксы, прозрачность, легенда по стилю, zoom-to-layer. Порядок — drag-n-drop.
- **Слой ЧС**: маркеры цвета уровня; кластеризация; фильтр-таймлайн снизу (период + play-анимация по датам); фильтры по виду/статусу/региону в FilterBar.
- **Инспектор** (справа): клик по объекту → карточка-peek (атрибуты, фото, связи, кнопка «Открыть полностью»). Мультивыбор рамкой — таблица выбранного.
- **Инструменты** (справа сверху): рисование (point/line/polygon — terra-draw) в редактируемый слой; измерение расстояния/площади; поиск по адресу/названию/координатам (geocoding по своим данным: admin_units+facilities, + парсер координат); экспорт видимой области в PNG (для докладов); шаринг «вида карты» (URL со стейтом слоёв/позиции).
- Редактирование геометрии объекта — по праву слоя, с историей (аудит + prev_geom в meta).
- Производительность: все слои — векторные тайлы Martin (таблицы `gis.*` авто-каталогом + SQL-функции для фильтрованных ЧС), генерализация на низких зумах.

## 5. Реестр ЧС (`/app/incidents`)

- DataTable: №, дата, вид (иконка+название), уровень (SeverityBadge), регион/район, статус, погибшие/пострадавшие, ущерб, ответственный. Фильтры: период, вид (дерево), уровень, статус, регион. Сохранённые фильтры. Экспорт выборки XLSX.
- **Создание донесения — форма ≤ 2 минут**: вид ЧС (быстрый поиск по дереву), дата/время (по умолчанию сейчас), место — клик по мини-карте ИЛИ поиск населённого пункта ИЛИ координаты; предварительные данные (жертвы — опционально), описание. Регион/район подставляются по точке. Кнопка «Создать и открыть карточку». Хоткей `C` из реестра.
- **Карточка ЧС** `/app/incidents/:id`: шапка (номер, вид, SeverityBadge, статус-степпер, кнопки: Добавить донесение / Изменить статус / Создать задачу / Создать канал / Связать документ). Вкладки: Обзор (карта-врезка + ключевые цифры + описание), Хронология (донесения + события системы), Силы и средства, Файлы (фото/акты), Связи, Обсуждение (comments).
- Изменение цифр жертв/ущерба — только через новое донесение (журналируемость).

## 6. Импорт/экспорт геоданных

- Импорт (`gis.import`): мастер — загрузка файла (SHP zip / GeoJSON / KML / GPKG / CSV с колонками координат) → worker `geo-import` (ogr2ogr → таблица `gis.l_<slug>`, репроекция в 4326, валидация геометрий) → предпросмотр (поля, число объектов, extent) → создание слоя в реестре + автостиль по типу геометрии. Ошибки — построчный лог.
- Экспорт: любой слой/выборка ЧС → GeoJSON / GPKG / SHP / CSV / XLSX (worker `geo-export`, уведомление со ссылкой).

## 7. Интеграция с QGIS / ArcGIS

- **QGIS → PostGIS напрямую** (основной путь): админ выдаёт учётку `gis_reader`/`gis_editor` (эндпоинт в админке создаёт PG-роль с паролем, доступ только к схеме `gis`, RLS не используем — гранулярность на уровне схемы). Подключение — по инструкции (host, 5432/TLS или VPN). Правки из QGIS попадают в те же слои и сразу видны в вебе (тайлы Martin без кэша либо с TTL 30с).
- **WMS/WFS (QGIS и ArcGIS)**: GeoServer, workspace `cuks`, слои публикуются из реестра автоматически (REST API GeoServer при `is_published_wms=true`). WFS-T разрешён для editor-учёток. URL: `https://host/geoserver/cuks/{wms|wfs}`. Basic-auth учётки GeoServer синхронизируются c платформенными (создаются админкой; v1 допускает отдельные учётки).
- Страница «Для ГИС-специалистов» в модуле: готовые URL, инструкции подключения QGIS/ArcGIS со скриншотами (статичный MD-контент).

## 8. Аналитика (`/app/analytics`)

**Дашборд «Оперативная сводка»** (главная страница платформы = `/app/dashboard`, часть этого модуля): за выбранные сутки/период — KPI-карточки (ЧС всего / погибшие / пострадавшие / эвакуированные / ущерб, с дельтой к пред. периоду), карта-врезка активных ЧС, лента последних донесений, «Требует внимания» (мои задачи/документы — агрегатор из других модулей).

**Дашборд «Статистика ЧС»**: фильтр-период+регион+вид; графики (ECharts): динамика по месяцам (bar+line), распределение по видам (treemap/bar), по регионам (bar + карта-хороплет по районам), тепловая карта день×час, топ-10 районов, жертвы/ущерб по видам. Каждый график — экспорт PNG; страница — «Печать» (print-css).

**Конструктор отчётов** (`analytics.build`): выборка = фильтры реестра ЧС + выбор колонок + группировки (по виду/региону/месяцу) с агрегатами (count, sum жертв/ущерба) → таблица → сохранить как отчёт (saved_filters) / экспорт XLSX (exceljs, с шапкой КЧС) / по расписанию на email (cron worker; v1 — ежемесячно).

**Типовые отчёты** (пресеты): «Сводка за сутки» (формат для доклада), «ЧС за период по видам», «Сравнение с АППГ» (аналогичный период прошлого года).

## 9. API (основное)

```
GET/POST /incidents  GET/PATCH /incidents/:id  POST /incidents/:id/reports
POST /incidents/:id/status  GET /incidents/:id/timeline  POST /incidents/export
GET /gis/layers  POST /gis/layers  PATCH /gis/layers/:id  DELETE ...
POST /gis/imports (file_id, options)  GET /gis/imports/:id
POST /gis/layers/:id/export  GET /gis/features?layer&bbox (инспектор)
PATCH /gis/features/:id (геометрия/атрибуты)  POST /gis/pg-accounts (админ)
GET /analytics/summary?period  GET /analytics/stats?filters  POST /analytics/query (конструктор)
GET/POST /analytics/reports (сохранённые)
```
Тайлы: `GET /tiles/{source}/{z}/{x}/{y}` — Martin, авторизация: подписанный короткоживущий query-token, выдаваемый api при загрузке карты (Martin за Caddy, проверка токена — Caddy forward_auth → api).

## 10. События и фоновые задачи
События: `incident.created|updated|status_changed` → уведомления по матрице (дежурные, руководство при sev≥3), аудит, канал ЧС. Jobs: `geo-import`, `geo-export`, `report-schedule`. WS: обновление карты/реестра (`incidents.updated`) для открытых клиентов.

## 11. Критерии приёмки
- Донесение создаётся за ≤ 2 мин, точка сразу на карте у всех открытых клиентов.
- Карта с 10k ЧС и 5 слоями держит 60fps на среднем ноутбуке (кластеризация+тайлы).
- Импорт SHP 100k объектов проходит; слой виден в вебе и в QGIS (PostGIS + WFS) без ручных действий.
- Дашборд статистики совпадает с контрольными SQL-запросами (тест на сид-данных).
- XLSX-отчёт открывается в Excel, кириллица/числа корректны.
- Все права: районный пользователь не видит чужой район в реестре (e2e).
