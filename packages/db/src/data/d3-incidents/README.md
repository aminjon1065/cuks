# D3 — ETL исторического реестра ЧС (1988–2020)

Артефакты фазы **D3** (`docs/plan/DATA-INTEGRATION.md`). Загружают по-инцидентные исторические реестры КЧС
в `app.incidents` + `app.incident_reports`, применяя кросс-волки D1 (гео) и D2 (типы).

## Файлы и запуск
- `stg-incidents.csv` — **4681 инцидент** (1988–2020), извлечён из 5 Excel-реестров детерминированным
  `../import/extract-incidents.py` (дедуп; standalone `Disaster 2018.xlsx` исключён как дубль).
- `../import/load-incidents.ts` — идемпотентный загрузчик (`pnpm --filter @cuks/db load:incidents`).

```bash
# предпосылки: seed-geo.sh (районы) + load:crosswalk (D1) уже выполнены
SRC=".../Data For CUKS/Всякая статистика/Disaster+" python3 packages/db/src/import/extract-incidents.py  # (пере)генерация CSV
pnpm --filter @cuks/db load:incidents
```

## Что делает загрузчик
1. Досеивает 4 кода справочника (`accident.*`, `nat.meteo.lightning`) — всехазардный реестр.
2. Стейджит `stg-incidents.csv` + `type-map.csv` (D2) в `stg`.
3. `INSERT…SELECT` в `app.incidents`: тип ← `type-map` (fallback `accident.other`); гео ← `stg.admin_alias`
   (район→регион), геокод к центроиду единицы (`ST_PointOnSurface`); `occurred_at` из года/месяца/дня
   (день-диапазон→первый, нет дня→15, нет месяца→июль, день клампится к длине месяца); `severity` — производная.
4. По одному `incident_report` на инцидент (снимок жертв/ущерба).

## Результат (проверено на живой БД)
- **4641 инцидент + 4641 отчёт** загружено (пропущено 40: 34 заграница — трудовые мигранты; 6 без года).
- **Вся геометрия валидна и в границах РТ**; 4512 привязаны к админ-единице, 129 (national/нераспознанные) —
  к центроиду РТ (region=null). Инварианты чисты (closed_at, reported≥occurred, severity 1–5).
- Топ-типы: сель 1187 · лавина 433 · землетрясение 429 · наводнение 410 · утопление 409 · ливень 394.
- По регионам: РРП 1417 · Согд 1031 · Хатлон 1010 · ГБАО 721 · Душанбе 333.

## Валидация против «10 сола» — расхождение объяснено
Годовые суммы **не сходятся** с агрегатом `10 сола 2013-2024.xlsx`, и это **не ошибка ETL**:
1. **Разный охват:** «10 сола» считает только **природные** ЧС (13 видов); реестр — всехазардный
   (утопления/ДТП/пожары ~40% записей за 2017–2020).
2. **Независимые компиляции:** даже природные-только не сходятся по годам — это разные документы с разной
   полнотой (2014: исходный файл почти пуст =12, «10 сола» =67; 2017: файл богаче =241, «10 сола» =94).
ETL **точно воспроизводит исходные по-инцидентные файлы** (2016: источник 152 → БД 149; 2014: 14 → 13).
«10 сола» остаётся отдельным аналитическим справочником (грузится агрегатами, решение D2/§3).

## Решения (зафиксировано)
- `id` импортных строк = `gen_random_uuid()` (v4) — bulk-insert на чистом SQL; id опаковый (FK/поиск), не влияет.
- `number` = `ЧС-{YYYY}-и{NNNN}` — не пересекается с живым `ЧС-{YYYY}-{seq}`.
- `source='report_doc'`, `status='closed'`, `created_by=null`. Заграница — пропуск; national — центроид РТ.
- Жертвы/ущерб — только структурно доступные (dead: 2018–2020; damage_est ~3%); текстовые жертвы/ущерб
  в `description`/`damage_note`. Точный парсинг из текста — D6.
