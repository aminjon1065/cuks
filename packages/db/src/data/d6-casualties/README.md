# D6 — жертвы из текста + агрегаты для валидации

Артефакты фазы **D6** (`docs/plan/DATA-INTEGRATION.md`). Дополняют реестр ЧС структурными жертвами,
извлечёнными из текстовых описаний, и дают агрегатный референс для валидации аналитики.

## 1. `casualties.csv` — погибшие/раненые/ущерб из описаний
В исторических реестрах 1988–2017 погибшие ЛЮДИ указаны только в свободном тексте описания (структурная
колонка `фавтида` есть лишь за 2018–2020, загружена в D3). Колонки: `number, dead, injured, damage_somoni, note`.

- **595 описаний** (упоминающих жертвы/ущерб) распарсены LLM-проходом со строгими правилами: считать ТОЛЬКО
  людей (не скот «голов КРС/МРС»), словоформы («погибла мать» = 1, «погиб ребёнок» = 1), исключать «пропал без
  вести», ущерб в сомони с конвертацией единиц (тыс.→×1000). Каждая запись — с `note`-обоснованием; критик
  перепроверил выбросы (53 и 55 погибших — реальные катастрофы 1988/1998, подтверждены).
- Результат: **270 инцидентов с погибшими, 46 с ранеными, 11 с суммой ущерба**.

> Регекс здесь ненадёжен (путает людей и скот, словоформы) — поэтому парсинг LLM. Применение — детерминированное
> (`load-casualties.ts`): значение пишется ТОЛЬКО где поле инцидента ещё 0/null (структурные данные 2018–2020 не
> затираются), затем пересчитывается `severity` и синхронизируются `incident_reports`.

### Запуск
```bash
pnpm --filter @cuks/db load:casualties   # после load:incidents; идемпотентно
```
Эффект (проверено на живой БД): инцидентов с погибшими **444 → 657**, всего погибших **≈509 → 1283**, с ранеными
**+46**, с суммой ущерба **+11**.

## 2. `stats-yearly.csv` — агрегаты «10 сола 2013-2024» (референс)
90 строк `year, type_code, count, source` — официальные годовые количества природных ЧС по видам за **2013–2023**
(вкл. 2021–2023, которых нет в по-инцидентных данных). Назначение — постоянный **референс валидации** дашборда/SQL.

**Важно (см. §D3):** эти агрегаты **не сходятся** с по-инцидентным реестром год-в-год — разный охват («10 сола» =
только природные ЧС) и независимые компиляции. Это ожидаемо; агрегаты используются как отдельный ориентир, а не
как строгий контроль сумм. Дашборд считает статистику из `app.incidents` напрямую.
