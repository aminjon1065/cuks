# Мониторинг — runbook администратора

Мониторинг платформы CUKS состоит из трёх слоёв (docs/08-deployment.md §Мониторинг, docs/modules/16 §7,
задача 7.3):

1. **Health-дашборд в админке** (`/app/admin/health`, право `admin.system.monitor`) — «взгляд одним
   глазом»: статусы сервисов, размеры БД/бакета/диска, очереди BullMQ, последний бэкап, ошибки за 24 ч.
2. **Uptime Kuma** — собственно мониторинг: периодические пинги сервисов и алерты.
3. **Алерты в чат-канал** — Uptime Kuma шлёт вебхук в API, тот постит системное сообщение в канал.

## 1. Health-дашборд

Открывается суперадмином/администратором платформы в разделе «Здоровье». Обновляется каждые ~15 c. Виджеты:

- **Сервисы** — пинги api→postgres/redis/minio/geoserver/martin/livekit/clamav. `Не настроен` — сервис не
  сконфигурирован (например, LiveKit без `LIVEKIT_URL`), это не падение.
- **Хранилище** — размер БД (`pg_database_size`) с разбивкой по схемам, объём и число объектов в бакете
  MinIO, свободное место на ФС контейнера api. Диск контейнера — лишь ориентир; реальный диск данных
  следит Uptime Kuma / хост.
- **Очереди задач** — waiting/active/delayed/failed/completed по каждой очереди BullMQ; при `failed > 0` —
  кнопка «Повторить» (перезапускает упавшие задачи очереди).
- **Последний бэкап** — по маркеру, который пишет `backup.sh` после успешного restic-снапшота.
- **Ошибки за 24 ч** — счётчик 5xx (почасовые корзины в Redis, суммируются за сутки).

Дашборд НЕ заменяет мониторинг — он не шлёт алертов и не хранит историю. Для этого — Uptime Kuma.

## 2. Uptime Kuma

Контейнер `uptime-kuma` уже в `compose.prod.yaml`. Его UI намеренно **не** выставлен наружу (порт слушает
только `127.0.0.1:3001`), чтобы консоль мониторинга не торчала в сеть. Настройка — через SSH-туннель.

```bash
# С рабочей машины админа:
ssh -L 3001:127.0.0.1:3001 <user>@<server>
# затем открыть http://localhost:3001 в браузере
```

При первом входе задать логин/пароль администратора Kuma (хранится в томе `uptimekuma`).

### Мониторы (Add New Monitor)

Kuma в сети `cuks_default`, поэтому обращается к сервисам по именам. Рекомендуемый набор:

| Тип | Имя | Проверка |
|---|---|---|
| HTTP(s) | API готовность | `http://api:3000/api/health/ready` — 200 когда всё ок; 503 при деградации зависимостей |
| HTTP(s) | MinIO | `http://minio:9000/minio/health/live` |
| HTTP(s) | GeoServer | `http://geoserver:8080/geoserver/web/` |
| HTTP(s) | Martin | `http://martin:3000/health` |
| TCP Port | PostgreSQL | `postgres:5432` |
| TCP Port | Redis | `redis:6379` |
| TCP Port | LiveKit | `livekit:7880` |
| TCP Port | ClamAV | `clamav:3310` |

Отдельно можно добавить дисковый монитор хоста (push-монитор + cron-скрипт `df`) — по вкусу.

## 3. Алерты в чат-канал

Uptime Kuma при падении/восстановлении монитора вызывает вебхук API, который постит системное сообщение в
заранее выбранный канал (например, «Дежурная смена» или служебный «Мониторинг»).

### Настройка API

1. Создать/выбрать чат-канал для алертов, скопировать его id.
2. В `.env` задать:
   - `MONITORING_ALERT_CHANNEL_ID=<id канала>`
   - `MONITORING_WEBHOOK_SECRET=<секрет>` (`openssl rand -hex 24`).
3. Перезапустить api: `dc up -d api`. Без обеих переменных эндпоинт `/api/monitoring/alert` отключён (404).

### Настройка уведомления в Kuma

Settings → Notifications → Add → тип **Webhook**:

- **Post URL**: `http://api:3000/api/monitoring/alert`
- **Request Body**: `Preset - application/json` (Kuma пошлёт `{ msg, monitor, heartbeat }`).
- **Additional Headers**:
  ```json
  { "X-Monitoring-Secret": "<тот же MONITORING_WEBHOOK_SECRET>" }
  ```

Привязать это уведомление ко всем мониторам (в каждом мониторе включить notification). Проверка — кнопка
«Test» в настройке уведомления: в канале должно появиться системное сообщение. Неверный/пустой секрет → API
отвечает 403 и сообщение не постится; при выключенном эндпоинте — 404.

> Безопасность: секрет сравнивается на сервере в постоянном времени; эндпоинт `@Public` (машинный вызов,
> без сессии/CSRF), единственный гейт — заголовок-секрет. Держите `MONITORING_WEBHOOK_SECRET` в `.env`
> (chmod 600), не логируйте.
