# 06. Дизайн-система и UX

Целевой уровень — Linear/Notion: спокойный, плотный, быстрый интерфейс без «госпортального» визуального шума. Пользователь — не айтишник; каждый экран должен быть понятен без обучения.

## 1. Принципы

1. **Одна главная задача на экран.** Второстепенное — в панели, меню «⋯», настройки.
2. **Статус всегда виден**: у документа, ЧС, задачи — цветной бейдж статуса в одном и том же месте (слева от заголовка).
3. **Действие ближе к объекту**: основная кнопка справа сверху карточки; массовые действия — в тулбаре при выделении.
4. **Скорость восприятия**: скелетоны (не спиннеры) на списках, optimistic UI в чате/задачах, переходы ≤ 200 мс.
5. **Пустое состояние = обучение**: иконка + одна фраза «что это» + кнопка первого действия.
6. **Плотность**: таблицы компактные (row 40px), формы и карточки — с воздухом (поля 36px, секции через 24px).
7. **Никогда**: градиенты, тени глубже уровня 2, более 2 акцентных цветов на экране, анимации дольше 250 мс, модалка поверх модалки.

## 2. Токены (CSS-переменные в `packages/ui/tokens`, семантика поверх Tailwind)

### Цвета — светлая тема (тёмная — те же роли, пересчитанные)
| Токен | Значение (light) | Использование |
|---|---|---|
| `--background` | #F8FAFB | фон приложения |
| `--surface` | #FFFFFF | карточки, панели |
| `--surface-2` | #F1F5F8 | вложенные области, hover строк |
| `--border` | #E2E8F0 | границы (1px, всегда) |
| `--text` | #0F172A | основной текст |
| `--text-muted` | #64748B | вторичный текст |
| `--primary` | #1256A0 | бренд-синий КЧС: кнопки, ссылки, активные состояния |
| `--primary-fg` | #FFFFFF | текст на primary |
| `--success` | #15803D | успех, «исполнено» |
| `--warning` | #B45309 | сроки «скоро», внимание |
| `--danger` | #B91C1C | ошибки, просрочено, деструктив |
| `--info` | #0369A1 | нейтральные подсказки |

### Уровни ЧС (фиксированная шкала, используется на картах, бейджах, дашбордах)
| Уровень | Токен | Цвет |
|---|---|---|
| Объектовый | `--sev-1` | #64748B (серый) |
| Местный | `--sev-2` | #CA8A04 (жёлтый) |
| Региональный | `--sev-3` | #EA580C (оранжевый) |
| Республиканский | `--sev-4` | #DC2626 (красный) |

### Прочее
- Радиусы: `--radius-sm 6px` (инпуты, бейджи), `--radius 10px` (карточки, кнопки), `--radius-lg 14px` (модалки).
- Тени: `--shadow-1` (0 1px 2px rgba(15,23,42,.06)), `--shadow-2` (0 4px 12px rgba(15,23,42,.10)) — максимум.
- Отступы: шкала 4px (4/8/12/16/24/32/48).
- Шрифт: **Inter** (self-hosted woff2: 400/500/600/700) + `system-ui` fallback; моно — `JetBrains Mono` для номеров/кодов.
- Типографика: 13px базовый UI, 14px тексты документов/чата, 16/20/24 заголовки (semibold), 12px подписи. line-height 1.5.
- Темы: light (по умолчанию) и dark (обязательна с фазы 0; дежурные работают ночью). Переключатель + `prefers-color-scheme`.

## 3. Каркас приложения (App Shell)

```
┌──┬────────────────────────────────────────────┐
│  │ Topbar: [назв. раздела/хлебные крошки] [🔍 Cmd+K] [🔔] [аватар] │
│ S├────────────────────────────────────────────┤
│ i│                                            │
│ d│              Контент раздела               │
│ e│                                            │
└──┴────────────────────────────────────────────┘
```

- **Sidebar** 240px, сворачивается до 64px (иконки+тултипы), состояние в localStorage. Пункты: Дашборд, Карта, ЧС, Аналитика, Документы, Задачи, Чат, Встречи, Файлы; ниже разделитель: Администрирование (по правам). Бейджи непрочитанного у Чат/Документы/Задачи. Внизу — аватар/имя/должность, переключатель темы.
- **Topbar** 56px: заголовок текущего раздела, глобальный поиск (открывает Command Palette), колокольчик уведомлений (поповер с лентой, «прочитать все»), меню профиля.
- **Command Palette (Cmd+K)**: поиск по всему (документы, ЧС, задачи, файлы, люди, каналы — сгруппировано) + быстрые действия («Создать донесение», «Новый документ», «Начать звонок»).
- Развёртка контента: страницы-списки на всю ширину; страницы-карточки — центр 960px; карта — fullbleed.

## 4. Компонентный слой (`packages/ui`)

База — shadcn/ui (Button, Input, Select, Dialog, Popover, Tabs, Toast, Tooltip, DropdownMenu, Sheet, Command, Table, Badge, Avatar, Skeleton, Calendar…). Поверх — собственные, обязательные к использованию:

| Компонент | Назначение |
|---|---|
| `DataTable` | Таблица на TanStack Table: сортировка, выбор строк, сохраняемые фильтры, колонки-настройка, пагинация, состояния loading/empty/error — всё встроено |
| `FilterBar` | Строка фильтров над таблицами: chips активных фильтров, «сбросить», сохранённые пресеты |
| `StatusBadge` | Бейдж статуса с единым словарём цветов по типам сущностей |
| `SeverityBadge` | Уровень ЧС по шкале sev-1..5 |
| `EntityLink` | Ссылка-чип на связанную сущность (иконка типа + номер + название) |
| `UserChip` / `UserPicker` | Аватар+имя+должность; выбор людей/подразделений с поиском |
| `OrgUnitPicker` | Дерево подразделений |
| `DateTimeField` | Дата/время в Asia/Dushanbe, быстрые пресеты («сегодня», «+3 дня») |
| `FileDropzone` + `AttachmentList` | Загрузка с прогрессом, список вложений с превью |
| `EmptyState` | Иконка + текст + действие |
| `PageHeader` | Заголовок + статус + действия страницы |
| `ConfirmDialog` | Деструктив: требует видимого имени объекта |
| `SidePanel` | Правая панель деталей (peek) для списков — карточка без ухода со страницы |
| `RichTextEditor` | TipTap: bold/italic/списки/ссылки/упоминания — для описаний и чата |

Правило: **новый UI собирается из этих компонентов**. Новый общий паттерн → сначала компонент в packages/ui, потом использование.

## 5. Паттерны

- **Список → деталь**: клик по строке открывает SidePanel (peek); двойной клик/enter — полная страница. У каждой карточки есть постоянный URL.
- **Формы**: одна колонка, max 560px; label сверху; required — «*»; ошибки под полем после blur; sticky-футер «Сохранить/Отмена» на длинных формах; автосохранение черновиков (документы, донесения) каждые 5с локально.
- **Мастера** (регистрация документа, импорт слоя): шаги сверху, «Назад» всегда доступно.
- **Уведомления-тосты**: успех 3с, ошибка — до закрытия, с кодом ошибки в «подробнее».
- **Числа и даты**: тысячные пробелом (12 500), даты `12.07.2026`, время `14:30`, относительное «5 мин назад» в лентах.
- **Клавиатура**: `Cmd+K` поиск, `C` — создать в текущем разделе, `Esc` — закрыть панель, `?` — шпаргалка хоткеев.

## 6. Карта (паттерны ГИС-UI)

- Полноэкранная карта; слева — панель слоёв (чекбоксы, прозрачность, легенда); справа — инспектор выбранного объекта; снизу — таймлайн-фильтр периода для ЧС; сверху-слева — поиск места/объекта; сверху-справа — инструменты (рисование, измерение, экспорт PNG, подложки).
- Кластеризация точек ЧС при отдалении; клик по кластеру — зум. Цвет маркеров = уровень ЧС, форма = статус (активная — пульсирующая точка).
- Тёмная подложка в тёмной теме.

## 7. Адаптивность

- Desktop-first (основная работа), брейкпоинты: ≥1440 комфорт, 1024–1440 норм, 768–1024 — сайдбар свёрнут, панели становятся оверлеями. <768 (телефон): полноценно — Дашборд, Чат, Задачи (просмотр), Уведомления, карточки документов (просмотр/согласование/подпись!); карта — просмотр без редактирования. Руководитель должен уметь согласовать и подписать с телефона.

## 8. Чеклист ревью каждого экрана (агент проходит перед завершением)

- [ ] Только токены (нет hex/px-магии в компонентах).
- [ ] Есть все состояния: loading (skeleton), empty (EmptyState), error (retry), нет данных из-за прав (403-экран).
- [ ] Тёмная тема — проверена визуально.
- [ ] Заголовок страницы, документ `title`, хлебные крошки корректны.
- [ ] Основное действие — одно, справа сверху; деструктив — через ConfirmDialog.
- [ ] Таблица: сортировка, фильтры, пагинация, выделение — через DataTable.
- [ ] Клавиатура: tab-порядок, Esc, Enter в формах.
- [ ] Тексты — из i18n, без обрезаний при длинных строках (русский длинный!).
- [ ] Скриншот двух тем приложить в PR/сессию.
