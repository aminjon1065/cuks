# Дорожная карта

Оценки — для одного разработчика с ИИ-агентом, полная занятость. Правило: **фаза не начинается, пока предыдущая не принята** по её критериям (кроме помеченных ∥ — можно параллелить). После фазы 2 платформа уже полезна (MVP) и может быть показана/внедрена пилотно.

| Фаза | Содержание                                                              | Оценка                | Зависимости          |
| ---- | ----------------------------------------------------------------------- | --------------------- | -------------------- |
| 0    | Фундамент: монорепо, инфраструктура, auth/RBAC, shell, дизайн-система   | 4 нед                 | —                    |
| 1    | Файловый менеджер (+ вложения как сервис)                               | 3 нед                 | 0                    |
| 2    | ГИС + реестр ЧС + аналитика (**основной модуль**)                       | 6 нед                 | 0, 1                 |
| 3    | Документооборот + ЭЦП                                                   | 6 нед                 | 0, 1                 |
| 4    | Задачи                                                                  | 3 нед                 | 0, 1                 |
| 5    | Чат                                                                     | 3.5 нед               | 0, 1                 |
| 6    | Звонки и записи                                                         | 3.5 нед               | 0, 5 (кнопки в чате) |
| 7    | Hardening: прод-деплой, бэкапы-drill, нагрузка, безопасность, полировка | 3 нед                 | все                  |
|      | **Итого**                                                               | **~32 нед (7–8 мес)** |                      |

Порядок 3↔4↔5 можно менять по требованиям заказчика; 2 — приоритет (основной модуль). Если нужен ранний вау-эффект: 0 → 1 → 2 → пилот.
