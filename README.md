# CUKS — Цифровая платформа КЧС

Единая корпоративная платформа Комитета по чрезвычайным ситуациям: аналитика и ГИС, документооборот с ЭЦП, файлы, чат, видеоконференции, задачи. Полностью self-hosted — работает на одном сервере КЧС, без внешних SaaS.

Этот репозиторий — **пакет проектной документации**, подготовленный для разработки силами одного разработчика с ИИ-агентом (Claude Code или аналог). Каждый документ написан как техническое задание, пригодное для прямой передачи агенту.

## Порядок чтения (для человека)

1. `docs/00-vision.md` — понять продукт.
2. `docs/plan/ROADMAP.md` — понять порядок работ и сроки.
3. `docs/01-architecture.md` + `docs/02-stack.md` — технические решения.
4. Остальное — по мере надобности.

## Как работать с ИИ-агентом

1. Открыть эту папку в Claude Code (или другом агенте). `CLAUDE.md` подхватится автоматически.
2. Взять стартовый промпт из `docs/plan/prompts.md` → «Kickoff».
3. Идти строго по фазам `docs/plan/ROADMAP.md`: одна сессия — один блок задач фазы.
4. После каждого блока: проверить результат руками (чеклист приёмки в конце каждой спеки модуля), закоммитить, попросить агента обновить `docs/plan/STATUS.md`.
5. Не давать агенту перепрыгивать фазы: фундамент (фаза 0) обязателен для всего остального.

Правила, которые агент обязан соблюдать, зафиксированы в `CLAUDE.md` — если агент отклоняется, укажите ему на конкретный пункт.

## Ключевые решения (кратко)

| Область           | Решение                                                                             |
| ----------------- | ----------------------------------------------------------------------------------- |
| Архитектура       | Модульный монолит: 1 SPA + 1 API + 1 worker                                         |
| Backend           | NestJS (Fastify), TypeScript strict, Drizzle ORM                                    |
| Frontend          | Vite + React 19 SPA, shadcn/ui + Tailwind, TanStack Query                           |
| БД                | PostgreSQL 17 + PostGIS, Redis 7                                                    |
| Файлы             | MinIO (S3-совместимое)                                                              |
| Видео             | LiveKit self-hosted + Egress (запись)                                               |
| ГИС               | PostGIS + Martin (векторные тайлы) + GeoServer (WMS/WFS для QGIS/ArcGIS) + MapLibre |
| ЭЦП               | Внутренний удостоверяющий центр, ECDSA P-256, штамп на PDF                          |
| Деплой            | Docker Compose + Caddy на одном сервере                                             |
| Пакетный менеджер | pnpm + Turborepo                                                                    |

Обоснования — в `docs/02-stack.md`.

## Требования к машине разработчика

Node 22 LTS, pnpm 9+, Docker + Docker Compose, 16 ГБ RAM (комфортно 32), ~30 ГБ диска под контейнеры и данные.

## Быстрый старт (разработка)

Требуется Node 22 LTS и Docker. Пакетный менеджер pnpm подключается через
Corepack (входит в Node) — глобальная установка не нужна.

```bash
# 1. Активировать pnpm (один раз на машину)
corepack enable pnpm

# 2. Установить зависимости
pnpm install

# 3. Поднять инфраструктуру (PostgreSQL+PostGIS, Redis, MinIO, ClamAV, maildev)
cp .env.example .env
pnpm infra:up            # = docker compose -f infra/docker/compose.dev.yaml up -d

# 4. Запустить все процессы (api:3000, web:5173, worker) параллельно
pnpm dev
```

Проверка здоровья API:

```bash
curl http://localhost:3000/api/health          # liveness
curl http://localhost:3000/api/health/ready     # readiness (PG/Redis/MinIO)
```

Swagger: <http://localhost:3000/api/docs>. Почта (maildev): <http://localhost:1080>.
MinIO-консоль: <http://localhost:9001>.

Проверки перед коммитом (то же гоняет CI):

```bash
pnpm typecheck && pnpm lint && pnpm test && pnpm build
```

Полезное:

```bash
pnpm infra:down          # остановить инфраструктуру
pnpm db:generate         # сгенерировать миграцию из схемы (drizzle-kit)
pnpm db:migrate          # применить миграции
pnpm db:seed             # сиды (реализуются в фазе 0.3)
```

## Статус

Фаза 0 (Фундамент) — в работе. Готовы задачи **0.1** (монорепо, tooling, CI) и
**0.2** (dev-инфраструктура, валидация env, health-эндпоинты). Прогресс и
принятые решения — в `docs/plan/STATUS.md`, чеклист — в `docs/plan/ROADMAP.md`.
