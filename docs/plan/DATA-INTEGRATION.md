## 1. Куда это ложится в текущей системе (факт по схеме БД)

Схема **реализована** (46 миграций, реестр ЧС с MVT-тайлами, geo-import/export через GDAL). Ключевое:
- `app.incidents` — `number`(uq), `type_code`, `severity 1–5`, `status`, `occurred_at`, `reported_at`(≥occurred),
  `region_id/district_id/jamoat_id`→`gis.admin_units`, `geom NOT NULL`(4326), `address_text`, `description`,
  `source∈{phone,report_doc,monitoring,other}`, `dead/injured/evacuated/affected`, `damage_est numeric(18,2)`, `damage_note`.
- `app.incident_reports` — снимок жертв/ущерба во времени (хронология донесений).
- `gis.admin_units` — **сейчас засеяны только 5 регионов** (ADM1) из `tj-admin1.geojson`. **Районов/джамоатов нет.**
- `gis.risk_zones` — `hazard_code`, `name`, `level 1–5`, `geom MultiPolygon`, `attrs jsonb`, `source`. **Пусто.**
- `gis.facilities` — инфраструктура. **Пусто.**
- `gis.layers` / `app.gis_imports` — реестр слоёв + мастер импорта (GDAL: SHP-zip/GeoJSON/KML/GPKG/CSV).
- Справочник `dictionaries.type='incident_type'` — **полное дерево видов ЧС с готовыми кодами** уже засеяно
  (`nat.geophys.earthquake`, `nat.geo.mudflow`, `nat.geo.landslide`, `nat.hydro.flood`, `nat.meteo.windstorm`,
  `nat.meteo.snowfall`, `nat.meteo.drought`, …).

**Готовая инфраструктура, которую переиспользуем (не изобретаем заново):**
- `infra/scripts/seed-geo.sh` — загрузка ADM1/2/3 из geoBoundaries (ogr2ogr → `gis.admin_units`, репроекция 4326, привязка родителей по геометрии). ⚠️ названия районов/джамоатов приходят **латиницей** (плейсхолдеры).
- `infra/scripts/build-basemap.sh` — офлайн-подложка PMTiles (Protomaps) по bbox Таджикистана.
- `apps/worker/src/queues/geo-import/*` — импорт через `gdal-async` (тот же движок, что `ogr2ogr`), `/vsizip/` для SHP-zip, репроекция в 4326, автоопределение типа геометрии, построчный лог ошибок.

### Три структурных пробела, которые надо закрыть перед загрузкой
1. **Нет районов/джамоатов** в `gis.admin_units` → инциденты и опасные зоны не к чему привязать точнее области.
2. **Нет русских названий** районов/джамоатов (geoBoundaries даёт латиницу) → нужен кросс-волк «латиница → RU/TG».
3. **Ни в одном источнике нет координат** → каждому инциденту нужна геометрия через геокодинг к центроиду
   района/джамоата (не точка события, а привязка к админ-единице — это осознанное упрощение, фиксируется).

### D2. Нормализация справочников — кросс-волки (1–2 дн)
Версионируемые файлы соответствий (CSV/TS), **проверяются заказчиком** перед прогоном:
- `type_map`: ОФАТХО → `incident_type.code`. Правило составных: выбираем доминирующий вид (сель > дождь),
  остальные — в `attrs.secondary_types`. Примеры:
  `сель/сели/СЕЛ→nat.geo.mudflow` · `землетрясение/Заминљунбї→nat.geophys.earthquake` ·
  `оползень→nat.geo.landslide` · `подъем уровня воды/БАЛАНДШАВИИ ОБ→nat.hydro.flood` ·
  `дожди/дождь→nat.meteo.downpour` · `ветер→nat.meteo.windstorm` · `лавина→nat.geo.avalanche` ·
  `снегопады→nat.meteo.snowfall` · `град→nat.meteo.hail` · `засуха→nat.meteo.drought` · `камнепад→nat.geo.rockfall`.
- `severity_rule`: производная 1–5 по порогам (погибшие / пострадавшие / сумма ущерба), т.к. явного уровня в данных нет.
- `date_rule`: YEAR(+Month+Day) → `occurred_at` (UTC, полдень Asia/Dushanbe). Нет дня → 15-е; нет месяца → 1 июля.
- **DoD:** каждый уникальный токен ОФАТХО имеет отображение; отчёт «нераспознанные типы» пуст либо согласован.

## 4. Риски и заметки
- Названия районов/джамоатов из geoBoundaries — латиница; без кросс-волка RU-названий карточки ЧС будут нечитабельны. **Кросс-волк — критический артефакт D1.**
- Составные события и разнонаписание — источник ошибок маппинга; отчёты «нераспознанного» на каждом прогоне обязательны.
- `.gdb`/`.ecw`/`.mxd` — форматы Esri; в пайплайн заходят только через ogr2ogr-конвертацию (`.gdb`→GPKG). `.mxd` не импортируется — только референс стилей.
- Идемпотентность всех ETL (повторный прогон не плодит дубли) — обязательна, по образцу существующих сидов.
- Права/аудит: массовая вставка — под системным актором, значимые события в `audit_log` по спецификации модуля.
