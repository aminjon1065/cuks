# 09. Безопасность

Госорган, данные о ЧС и служебные документы. Модель нарушителя: внешний атакующий из интернета; инсайдер с ограниченными правами; утерянное устройство сотрудника. Не в модели v1: целевые APT, компрометация самого сервера.

## 1. Базовая гигиена (обязательно, фаза 0)

- Вся валидация входа — zod на каждом эндпоинте; параметризованные запросы (Drizzle) — SQL-инъекции исключены by design; никакого сырого HTML (React), rich-text — через whitelist-схему TipTap JSON, серверная санитизация при рендере в PDF/email.
- Заголовки: CSP (`default-src 'self'`; connect-src self + livekit; img/media self + blob; без unsafe-inline — Vite build без инлайнов), HSTS, X-Frame-Options DENY, nosniff, Referrer-Policy same-origin.
- Cookies: httpOnly, Secure, SameSite=Lax + CSRF-заголовок на мутации + проверка Origin.
- Rate-limits (per-IP и per-user, Redis): `/auth/*` 10 rpm; мутации 120 rpm; чтение 600 rpm; загрузки 20/мин; отдельно — глобальный поиск 60 rpm. 429 с Retry-After.
- Секреты: только env/volume; в логах — редактирование (pino redact: authorization, cookie, password, totp).
- Права PG: роль приложения без DDL (миграции — отдельной ролью), без DELETE на audit-схему. Отдельные роли `gis_reader`/`gis_editor` для QGIS-доступа — только схема `gis`.
- Обновления образов — ежемесячный regламент; `docker scout`/trivy — в CI.

## 2. Загрузки файлов

- Лимиты: файл ≤ 2 ГБ (конфиг), запросы presign — с квотой пользователя.
- MIME-sniffing по магическим байтам (не доверять расширению); опасные типы (exe, bat, js, SVG со скриптами…) — помечаются и скачиваются только как attachment.
- ClamAV: каждый новый объект → очередь `av-scan`; до вердикта файл `pending` (скачивание разрешено с предупреждением — конфиг; для ДСП-контуров можно запретить), `infected` → карантин, уведомление админу.
- Изображения — перекодирование sharp (EXIF-очистка) для превью/аватаров.
- Отдача только через presigned GET с `Content-Disposition`; прямой листинг бакетов закрыт.

## 3. Конфиденциальность (гриф ДСП)

- У документов и файлов — поле `confidentiality: normal|dsp`. ДСП видят только обладатели `docflow.confidential.view` **и** входящие в допуск-список карточки.
- Каждое открытие/скачивание ДСП — запись в `audit.read_log`.
- ДСП исключаются из глобального поиска для не-допущенных; превью не генерируются; в уведомлениях — только номер, без темы.

## 4. Внутренняя ЭЦП — крипто-спецификация

**Цель**: юридически значимая внутри организации подпись (целостность + авторство + время), с последующей возможностью добавить государственную квалифицированную ЭЦП как второй тип.

### УЦ (CA)
- Корневой ключ: ECDSA P-384, генерируется скриптом при установке, хранится в volume `ca_data` (доступен только api), пароль-парольная фраза в env. Само-подписанный сертификат CA, срок 10 лет.
- Сертификаты пользователей: ECDSA P-256, срок 2 года, subject = username + full_name + должность, выдаются автоматически при активации подписи пользователем. Отзыв — админом (таблица `certificates`: user_id, serial, pubkey, not_before/after, revoked_at, reason).

### Ключи пользователей
- Генерируются **в браузере** (WebCrypto, `extractable: false`), приватный ключ хранится в IndexedDB устройства. CSR → api → сертификат CA.
- Устройств может быть несколько — каждый со своим сертификатом (kind=device). Потеря устройства = отзыв сертификата; старые подписи остаются проверяемыми (сертификат хранится).
- Подписание требует подтверждения: повторный ввод пароля или TOTP (если включён) — «осознанное действие».

### Подпись
- Подписывается: SHA-256 конкретной **версии файла** + канонический JSON реквизитов карточки (номер, дата, тема) — итоговый digest.
- Формат хранения: `signatures`: doc_version_id, user_id, certificate_id, algorithm `ECDSA_P256_SHA256`, payload_hash, signature (base64), signed_at (серверное время), context (`approve|sign|acknowledge`).
- Проверка: эндпоинт + страница `/verify/<signature_id>` (доступна всем аутентифицированным): валидность подписи, цепочка до CA, статус отзыва на момент подписи, хэш совпадает с текущей версией файла.
- **Штамп на PDF** при экспорте: последняя страница-приложение (pdf-lib): таблица подписей (кто, должность, когда, серийник сертификата) + QR-код на страницу проверки. Оригинал файла не модифицируется — штампованный PDF это отдельный артефакт «выгрузка с отметками».

### Явно НЕ делаем
- Собственные крипто-примитивы; хранение приватных ключей пользователей на сервере; XAdES/CAdES-контейнеры (v2 — при интеграции гос-УЦ).
